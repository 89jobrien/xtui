# Plan: xtui v1 — Multi-Project Dev Dashboard

**Status:** Done (v0.1.0)

## Goal

Transform xtui from a single-project xtask runner into a multi-project dev
dashboard that discovers, organizes, and runs project commands from 7 sources
with history, search, output persistence, and command chaining.

## Context Map

### Files to Modify

| File              | Purpose                | Changes Needed                                             |
| ----------------- | ---------------------- | ---------------------------------------------------------- |
| `src/app.rs`      | App state + event loop | Multi-project, tabs, search mode, pipeline state           |
| `src/ui.rs`       | TUI rendering          | Tabbed sources, status bar, search overlay, project picker |
| `src/discover.rs` | Xtask command parsing  | Extract into `XtaskSource` impl, keep `parse_source`       |
| `src/runner.rs`   | Process spawning       | Add pipeline runner (sequential commands)                  |
| `src/main.rs`     | Entry point            | Use registry for project resolution                        |
| `Cargo.toml`      | Dependencies           | Add serde, serde_json, toml, dirs                          |

### New Files

| File              | Purpose                                   |
| ----------------- | ----------------------------------------- |
| `src/source.rs`   | `CommandSource` trait + 7 implementations |
| `src/registry.rs` | Project discovery, JSON cache, favorites  |
| `src/history.rs`  | Run history + output persistence          |
| `src/pipeline.rs` | Sequential command chaining               |
| `src/search.rs`   | Output search state + match tracking      |
| `src/status.rs`   | Git status collection for dashboard       |

### Dependencies (consumers of changed APIs)

| File          | Relationship                                                          |
| ------------- | --------------------------------------------------------------------- |
| `src/ui.rs`   | Consumes `App`, `Focus`, all new state types                          |
| `src/app.rs`  | Consumes `CommandSource`, `Registry`, `History`, `Pipeline`, `Search` |
| `src/main.rs` | Consumes `Registry` for startup                                       |

### Test Coverage

| Test                          | Covers                                     |
| ----------------------------- | ------------------------------------------ |
| `src/discover.rs` (inline)    | `parse_source` — xtask command extraction  |
| `src/discover.rs` (proptests) | Fuzz `parse_source` for panics/idempotency |
| `src/app.rs` (inline)         | `next()`/`previous()` navigation           |
| `src/runner.rs` (inline)      | `spawn_command` output capture             |

### Risk

- [x] `XtaskCommand` is `pub` — renamed/moved, all consumers in `app.rs`
      and `ui.rs` must update imports
- [x] `discover_commands` is `pub` — replaced by `CommandSource` trait,
      callers in `app.rs` and `main.rs` must update
- [x] New deps: `serde`, `serde_json`, `toml`, `dirs` — all well-known,
      no supply chain concern
- [x] `App` struct gains many fields — existing tests in `app.rs` need
      updated constructors

## Tech Stack

- Rust 2024 edition
- New deps: `serde` + `serde_json` (config/history), `toml` (mise),
  `dirs` (XDG config path)
- Existing: `ratatui`, `tokio`, `anyhow`, `regex`

   pub struct SourceCommand {

       pub source: String,

   ```rust
   pub struct XtaskSource;

   impl CommandSource for XtaskSource {
       fn name(&self) -> &str { "xtask" }

       fn discover(&self, project: &Path) -> Result<Vec<SourceCommand>> {
           let main_path = project.join("xtask/src/main.rs");
           if !main_path.exists() {
               return Ok(vec![]);
           }
           let source = std::fs::read_to_string(&main_path)?;
           let cmds = crate::discover::parse_source(&source);
           Ok(cmds
               .into_iter()
               .map(|c| SourceCommand {
                   name: c.name,
                   description: c.description,
                   source: "xtask".into(),
               })
               .collect())
       }
   }
   ```

4. Register module in `main.rs`: `mod source;`

5. Verify:

   ```

   cargo clippy → zero warnings
   ```

6. Commit: `feat(xtui): add CommandSource trait and XtaskSource`
